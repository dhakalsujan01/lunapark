About Us → Park history, mission, trust-building info

Contact → Google Maps embed, inquiry form, call/email/social buttons

🎟️ Booking & Ticket System (Login Required)

User must login/signup (NextAuth) to buy tickets.

Stripe checkout integration → only after verified payment via webhook signature tickets are issued.

Each ticket = unique signed QR code.

Ticket validation flow:

Staff scans QR → API checks if valid → if valid, mark used → becomes invalid after scan.

Tickets can be single-ride or package-based.

Users can view their purchased tickets in dashboard.

👥 Roles & Access

User → signup/login, buy tickets, view their QR codes.

Ticket Checker → QR scanner panel, validate tickets (mark used).

Admin → full access to manage rides, packages, users, orders, SEO, settings.

🔐 Security Rules

All sensitive APIs protected with role-based middleware.

Stripe webhook must verify signature header.

Tickets only generated after verified Stripe event.

Tickets invalid after first use.

Forgot password flow with expiring tokens (hashed in DB).

CSRF protection, input validation, rate limiting, secure cookies, env secrets.

Passwords & tokens never stored in plaintext.
